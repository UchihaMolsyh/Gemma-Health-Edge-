# Gemma Health Edge (Flutter)

A **mobile health companion** that runs on your phone and can chat with:

- A **local** (or tunneled) OpenAI-compatible server (for example `llama-server` on your PC)
- A **cloud model provider** via **OpenRouter** (API key required)

This repo contains the full Flutter app for **Android + iOS**, including:

- **Streaming chat UI**
- **Image input** (gallery / camera)
- **Voice input** (speech-to-text) and **voice output** (TTS)
- **Local RAG** over CSV datasets in `assets/datasets/`
- **Watch / wearable metrics** (via HealthKit / Health Connect, plus optional BLE)
- **Health calendar** (lightweight tracking stored locally)
- **7-language-ready localization** (Flutter `generate: true`)

> Medical disclaimer: This app provides **general health information only** and is **not** a substitute for professional medical advice, diagnosis, or treatment.

---

## Why this is "Edge"

Gemma Health Edge is designed around a simple idea:

- The **phone app** is the fast, friendly, sensor-adjacent interface.
- The app can use a **local runtime** (PC on LAN / USB tethering / tunnel) *or* a **cloud provider** (OpenRouter).
- Your **local knowledge** (CSV datasets) stays on-device and can be injected as context.

## How the app talks to the model

The chat runtime is OpenAI-compatible. In code, requests are sent to:

- **Local server mode**: `POST {serverUrl}/v1/chat/completions`
- **Cloud mode (OpenRouter)**: `POST https://openrouter.ai/api/v1/chat/completions`

The app supports:

- **Token streaming** (SSE)
- **Text + image prompts** (image is base64-encoded; support depends on the selected model/provider)

---

## Connection options

### Option A: Local server on the same Wi‑Fi / LAN

1. Start your server on the PC with host bound to all interfaces (e.g. `0.0.0.0`).
2. Find the PC's LAN IP.
3. In the app settings, set server URL to something like:

```text
http://192.168.x.x:8080
```

### Option B: USB tethering (low latency, no Wi‑Fi)

1. Enable **USB tethering** on Android.
2. Point the app at the tether gateway IP/port used by your setup.

### Option C: Tunnel (remote access)

If you run a tunnel (e.g. Cloudflare Tunnel) in front of your server, you can use the HTTPS URL in the app.

### Option D: Cloud API (OpenRouter — no PC required)

1. Enable `Use Cloud API (No PC)` in Settings.
2. Paste your **OpenRouter API key**.
3. Choose a **Model ID** (default in code is `google/gemma-2-27b-it`).

## Local RAG (CSV datasets)

The app ships with CSV files under `assets/datasets/` (for example: `conditions.csv`).

At runtime:

- The CSVs are **loaded from app assets**
- A **keyword/substring retrieval** step selects relevant rows
- The retrieved snippets are injected into the prompt as context

This gives you:

- **Faster** answers for common topics
- **Local context injection** even if you're using a cloud model

Note: the **model response still requires a reachable model backend** (local server or cloud). The CSV retrieval itself is local, but it does not replace the model.

## Watch & wearable metrics

The app includes a **Health Metrics** screen (`/health`) that can read and store wearable/device measurements.

### Mode 1: OS health store (recommended)

This is the most compatible approach.

- **iOS**: reads from **Apple Health / HealthKit** (Apple Watch typically syncs here)
- **Android**: reads from **Health Connect** (Wear OS / vendor apps typically sync here)

Supported types (best-effort depending on what your device provides):

- Steps
- Heart rate
- Blood oxygen (SpO₂)
- Blood pressure (systolic/diastolic)

### Mode 2: Direct Bluetooth LE (optional)

You can also connect directly to BLE devices that expose **standard GATT services**:

- Heart Rate Service (`0x180D`)
- Blood Pressure Service (`0x1810`)
- Pulse Oximeter Service (`0x1822`)

This mode is **not guaranteed** to work with every smartwatch because many watches use proprietary protocols.

### Storage

All collected measurements are stored **locally on the phone** (Hive box `health_metrics`).

---

## Privacy & data storage

Designed to be conservative by default:

- **Local storage** is used for sessions, settings, and calendar data.
- The server URL / cloud mode is user-controlled.
- The **OpenRouter API key** is stored using `flutter_secure_storage` (Keychain on iOS, EncryptedSharedPreferences on Android).
- When "research" is enabled, the app may call Wikipedia for summaries.

If you plan to use a tunnel/public URL or cloud mode, treat it like exposing a private health interface:

- Prefer authenticated tunnels / access control.
- Avoid logging prompts/responses on the server.

## Troubleshooting

### The app cannot connect to the server

- Verify the server is reachable from the phone (same Wi‑Fi / tethering).
- Make sure the server binds to `0.0.0.0` (not `127.0.0.1`).
- Confirm the URL includes the correct port.

### Streaming responses don't appear

- Confirm your server supports streaming for `/v1/chat/completions`.
- Try non-stream mode (if you have a toggle) to isolate the issue.

### Images don't get processed

- Your server/model must support multimodal input.
- Verify the server's expected payload format for image content.
